# CANOPY Data Handoff

This handoff is for teammates building the backend and frontend while the data
lane remains independent.

## Contract

Every scenario line is a canonical `Signal`:

```txt
id, ts, domain, source, realism, confidence, location, payload, provenance
```

The backend owns:

```txt
Signal -> Anomaly -> Attribution -> Decision -> UIEvent
```

The frontend should not consume `Signal` directly for the final demo unless it
is in fallback mode. It should consume `UIEvent`.

## Scenario Order

Run the beats in this order:

1. `scenarios/beat1.jsonl`
2. `scenarios/beat2.jsonl`
3. `scenarios/beat4.jsonl`
4. `scenarios/beat47.jsonl`

Fast replay example:

```bash
uv run python scripts/replay.py scenarios/beat2.jsonl --speed 20 --post-url http://localhost:8000/signals
```

Dry-run example:

```bash
uv run python scripts/replay.py scenarios/beat47.jsonl --dry-run
```

## Expected UI Events

`data/expected_ui_events.json` gives backend/frontend a shared target for the
demo beats. It is not generated by the backend yet; it is a fixture describing
what the downstream pipeline should emit after fusing groups of Signals.

Use it for:

- Backend deterministic fusion tests.
- Frontend fallback demo data.
- Demo script timing.

## Local WebSocket Bridge

The experimental bridge lives at `canopy.api:app`.

Run it with:

```bash
python -m uvicorn canopy.api:app --host 127.0.0.1 --port 8000
```

It exposes:

```txt
POST /signals
WS   /ws/brigade
```

`POST /signals` validates a canonical `Signal`, publishes it to
`signals.{domain}`, and feeds the backend pipeline. `/ws/brigade` streams:

```txt
{ "type": "signal", "data": Signal }
{ "type": "ui_event", "data": UIEvent }
```

The frontend uses `VITE_CANOPY_WS_URL`, defaulting in dev to:

```txt
ws://127.0.0.1:8000/ws/brigade
```

## KB Seed Entries

`data/kb_seed_entries.json` maps scenario signal IDs to public-source-backed
threat concepts. The attribution agent can retrieve these entries by
`scenario_signal_ids`, `domain`, or `capability_type`.

Important language rule:

```txt
Say "consistent with" or "confidence-scored assessment"; do not say "proves".
```

## Realism Labels

- `real_source`: public/citable data or public-style cache.
- `mock_operational`: synthetic operational inject.
- `synthetic_orbital_overlay`: fictional space object or maneuver overlay.

The UI should surface synthetic/fictional status only when needed for judge
transparency, not as operational clutter.
